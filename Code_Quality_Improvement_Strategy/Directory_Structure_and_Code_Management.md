##  📝 개요
프로젝트가 커질수록 디렉터리 구조가 중요해짐. 코드의 확장성과 유지보수를 위해 효울적인 프로젝트 구조를 관리하는 방법에 대해 학습한 내용을 정리.

## 1. 🖍️ 코드 관리 방식의 중요성
코드를 작성하다 보면 다양한 모듈(컴포넌트, 컨테이너, 커스텀 훅, 유틸리티 함수 등)을 여러 파일로 나누게 된다. 이러한 파일들을 효율적으로 배치하는 디렉터리 구조는 다음과 같은 이유로 중요하다.
1. 참조 관계 명확화:
   - 코드 의존성을 직관적으로 파악할 수 있어 참조해야 할 파일과 그렇지 않은 파일을 구분할 수 있음.
2. 유지 보수 용이성:
   - 연관된 코드가 한 디렉터리에 묶여 있으면 삭제나 수정이 용이함.
3. 코드 응집도 유지:
   - 기능별로 파일을 그룹화하여 응집도를 높이고, 불필요한 코드 참조를 방지함.
  
## 2. 📉 기존 방식의 문제점
🚨 모듈 종류에 따라 분류된 디렉터리 구조
```
└─ src
   ├─ components
   ├─ constants
   ├─ containers
   ├─ contexts
   ├─ remotes
   ├─ hooks
   ├─ utils
   └─ ...
```
### ❗️ 문제점
1. 의존 관계 파악이 어려움: 
   - 모듈 종류별로 나누면 어떤 코드가 어떤 코드를 참조하는지 명확하지 않음
2. 코드 응집도 저하: 
   - 서로 연관된 코드가 분리되어 관리되므로 유지보수가 어려움
3. 불필요한 코드 남음: 
   - 특정 기능이 삭제되었을 때 관련 코드가 누락될 가능성이 큼
  
## 3. 📈 개선된 디렉터리 구조
### ✅ 도메인 중심의 디렉터리 구조
```
└─ src
    ├─ common // 공통 컴포넌트 및 유틸리티
    │   ├─ components
    │   ├─ containers
    │   ├─ hooks
    │   ├─ utils 
    │   └─ ...
    │
    └─ domains
        │
        ├─ Domain1  // Domain1에서만 사용되는 컴포넌트 및 유틸리티
        │     ├─ components
        │     ├─ containers
        │     ├─ hooks
        │     ├─ utils
        │     └─ ...
        │
        └─ Domain2  // Domain2에서만 사용되는 컴포넌트 및 유틸리티
                ├─ components
                ├─ containers
                ├─ hooks
                ├─ utils
                └─ ...
```
### 🔖 장점
1. 코드 응집도 향상:
   - 관련된 코드가 한곳에 모여 있어 유지보수가 편리함
   - 예: Domain1의 경우 관련 컴포넌트, 컨테이너, 훅, 유틸 등 한 디렉터리 위치
2. 의존 관계 파악 용이:
   - 잘못된 코드 참조를 쉽게 인지 할 수 있음
   - 예시:
        ``` js
        // ❌ 잘못된 참조
        // Domain1의 컴포넌트에서 Domain2의 훅을 직접 참조
        import { useProject } from '../../Domain2/hooks/useProject';

        // ✅ 올바른 참조
        // 같은 도메인 내 훅 사용
        import { usePost } from '../hooks/usePost';

        // 다른 도메인의 기능이 필요하다면 common을 통해 공유
        import { useAuth } from '../../common/hooks/useAuth';
        ``` 
3. 불필요 코드 삭제 용이
   - 특정 기능 관련 코드가 한 디렉터리에 묶여 있어 관리가 쉬움.
   - 예: Domain1 기능 제거 시 /domains/Domain1 디렉터리만 삭제

## 4. 💻 실제 적용 사례
```
└─ domains              // 도메인별 구조
    └─ post            // 게시글 도메인
    ├─ components   // UI 컴포넌트
    │  ├─ PostList.jsx
    │  ├─ PostDetail.jsx
    │  └─ PostEditor.jsx
    ├─ containers  // 게시글 관련 비즈니스 로직
    │  ├─ PostListContainer.js
    │  ├─ PostDetailContainer.js
    │  └─ PostEditorContainer.js
    ├─ hooks       // 게시글 관련 훅
    │  ├─ usePost.js
    │  └─ useComments.js
    └─ services    // 게시글 API 서비스
        └─ postApi.js   
```
## 5. 🔑 Key Points
- 디렉터리 구조는 코드의 확장성과 유지보수성을 결정하는 중요한 요소
- 도메인 중심 구조를 통해 코드의 응집도 향상 및 의존성 명확
- 팀원들과의 협업에 수월함
